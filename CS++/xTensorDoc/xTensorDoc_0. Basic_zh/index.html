    示例

    流形和張量場

    從定義一個流形及其切叢上的張量場開始：

    DefManifold[M3, 3, {a, b, c, d, e}]
(*定義了流形的記號、維數和其使用的指標*)

    定義流形的同時定義了其上的切叢，很多量都會自動定義、自動計算，當定義或計算它的條件滿足的時候。

    DefTensor[v[a], M3]
DefTensor[F[-a, -b], M3, Antisymmetric[{-a, -b}]]
(*定義該流形上的矢量和(0, 2)型反對稱張量*)

    縮並/作用/張量積（我無意混淆之）變得如此簡單：

    F[-a, -b] v[b] v[a]
Simplification[%]

    化簡必需顯示地要求，\(\(F_{ab}v^av^b=F_{[ab]}v^{(a}v^{b)}=0\)\)。該函數可以識別同一量的不同指標形式，如 \(v_cv^c\) 和 \(v_bv^b\)。

    協變導數

    這個東西有很多的選項，我們先查看一下：

    Options[DefCovD]
(*-->
{SymbolOfCovD -> {";", "\[EmptyDownTriangle]"}, Torsion -> False, ...}(*後面的東西未展示出*)
*)

    可以看到，默認地有分號和 \(\nabla\) 記號，以及默認流形 🌠 有曲率而無撓率。

    DefCovD[Cd[-a]](*衆所周知，倒數算符很像一個對偶矢量*)

(*已經自動有 Leibnitz 法則，不過這不算什麽：*)
Cd[-a][F[-b,-c]v[c]](*注意這個命令的書寫方式，例如：Cd[-a]@v[b]*)

(*更要緊的是他已經自動定義了帶有正確關係的各種導出量：*)
RiemannCd[-a,-b,-c,d]+RiemannCd[-b,-a,-c,d]//Simplification

    展示一些操作

    (*不用背公式啦*)
RiemannCd[-a,-b,-c,d]//RiemannToChristoffel

(*對某幾個指標反對稱化，再利用 Riemann 張量的性質進行化簡*)
Antisymmetrize[Cd[-e][RiemannCd[-c,-d,-b,a]],{-c,-d,-e}]//Simplification

(*利用 Riemann 張量定義交換導數算符*)
CommuteCovDs[Cd[-a]@Cd[-b]@v[c],Cd,{-b,-a}]//ScreenDollarIndices

    度規及其導出量

    (*定義度規，同時指定與它相適配的導數算符*)
DefMetric[-1,g[-a,-b],CD,SymbolOfCovD->{"|","D"}]
(*你會發現系統自動幫你定義了一堆東西 ❤️*)

(*真的相適配嗎？：*)CD[-a][g[-b,-c]]

    顯示地要求縮並：ContractMetric，將倒數算符排序：SortCovDs。

    李導數和李括號

    LieD[3v[a]][F[-a,-b]v[b]]
Bracket[v[a],F[a,-b]v[b]][a]

    清除定義

    玩膩了，毀滅吧，但是要一步一步地 🤨。

    UndefMetric[g]
UndefTensor/@{v,F};
UndefCovD[Cd]
UndefManifold[M3]

    總結

    總結所有可以定義在 xTensor 的對象（原作者在此提及了一些合作者或貢獻者，對其表示感謝）：

    • 流形
    • 映射
    • 切叢
    • （不同類型的）指標
    • 張量、尤其是度規
    • 協變導數算符、包括偏導數
    • 常量符號
    • 參數
    • ？（譯者還不清楚，下同）
    • 標量函數
    • 坐標系和基（需使用包 xCoba）

    已經預定義的運算：

    • 李導數
    • 李括號
    • ？
    • 變分

    已經預定義的操作： - 索引对象之间的关系和规则 - 求解指標方程 - 表达式的规范化和简化 - 自动定义曲率张量，以及它们之间的关系 - 復共軛

    已經預定義的、在 GR 中常見的算法/執行程序： - Warped metric decomposition(?) - ADM-like decomposition(?) - 廣義相對論中的攝動理論（需使用包 xPert） - 球面張量諧波（需使用包 Harmonics） - 操作黎曼张量不变量（需使用包 Invar） - 旋量計算

    警示

    譯者直譯作者原文：

    xTensor 对于普通的 Mathematica 用户来说是一个非常强大的工具，但是对于一个没有经验的 Mathematica 用户来说可能很难。这是因为我并没有试图隐藏 Mathematica 的微妙之处(比如延迟赋值，独特的符号或增值值)，而是尽可能多地使用这些强大的工具。如果你不理解这三个列出的概念，请花一些时间(不是很多)在 Mathematica 教程上。即使最终不使用 xTensor，它也会得到回报。

    如果你认为你可以用几行就能完成你的张量计算，那么 xTensor 可能就太多了。这个系统被设计成在开发大型项目时是高效的，而不是小型计算:一个通用的张量计算机代数系统总是有数百个命令，而且只有大型项目才会给你足够的动力去学习它们。

    xTensor 是一个抽象表达式的操纵符。它不能计算(至少不容易)黎曼张量的分量与一个给定的度規分量矩阵相关，诸如此类。xTensor 专门用于处理具有某些性质(比如反对称)的对象族所服从的恒等式。许多用户在开始时感到失望，因为他们期望的是分量类型的计算。使用双包 xCoba。
