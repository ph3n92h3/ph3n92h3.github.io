    Hexo&NexT

    這個文檔用來記錄使用 Hexo 搭建和日常維護博客時的 tips。

    2022.02.04

    今天初次嘗試並最終搭建出來一個還不錯的博客，事實上，官方的教程是非常詳細的：

    https://hexo.io/zh-cn/docs/

    https://theme-next.iissnan.com/

    我只記錄一下今天遇到的坑 🕳。

    1. 開始之前~
    2. 爲什麽我沒有使用 Jekyll？因爲這個東西的使用依賴於 Ruby 語言，而這個語言的版本管理方式和包版本管理我實在是閙不機密，裝了各種版本的 Ruby 都不能按照教程的步驟走下來，出現的報錯我看懂了，但是它似乎自相矛盾而不存在一個合適的版本組合……
    3. 爲什麽我剛開始準備使用 Jekyll？因爲我是個原教旨主義者，GitHub 官方推薦了它……
    4. 安裝 Node.js and Git 沒有什麽坑。
    5. 安裝 NexT 主題：官方文檔中的命令是過時的，新的為：git clone https://github.com/theme-next/hexo-theme-next themes/next
    6. 似乎沒什麽人刻意地提及這一點：Hexo 推薦的方式似乎是需要在 Git 上準備兩個倉庫，一個用來放置源碼，似乎一般叫做 blog ；另一個則是 <username>.github.io，因爲 Hexo 有一個“一鍵部署”的功能，它先生成靜態網頁，之後在將這組靜態網頁覆蓋到 <username>.github.io 倉庫中。當然這兩個倉庫之間需要聯係：在站點配置文件末尾寫入如下内容（細節部分不同人寫的教程互異，這裏是官方的寫法）：
    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: https://github.com/<username>/<username>.github.io
  branch: master
    1. 調試（一邊寫一邊在本地看看寫出來的效果）：hexo s --debug
    2. 部署與發佈：
    hexo clean
hexo g
hexo d  #注意此時不挂 VPN 的情況成功率較高 :)
    1. 語言設置：繁體中文一律設置 zh-TW。注：本人堅持“一個中國”原則，堅決反對任何企圖分裂中國的行爲。
    2. Hexo 和 NexT 的官方文檔都說可以優雅地使用 {% ... %} 這種方式創建環境或引用，但是我這樣做似乎還沒有奏效
    3. 關於 MathJax 的使用，要在主題配置文件相應的配置項中修改：
      per_page: false   # 這裏為 false 時即爲每一頁都使用 MathJax
  # 我踩的坑我就不説了……
  ...
  mathjax:
    enable: true    # 即啓用
    1. 設置背景圖片及文字透明：圖片為：./themes/next/source/images/background.jpg；新建配置文件 ./source/_data/styles.styl 並寫入：
    // background

body {
  background:url(/images/background.jpg);
  height:100%;
  width:100%;
  background-repeat:repeat-x;
  background-attachment:fixed;
  background-size:100% 100%;
}

// page-opacity

.content-wrap {
  opacity: 0.8;
}
.sidebar {
  opacity: 0.8;
}
.header-inner {
  background: rgba(255,255,255,0.8);    //這裏不能用 'opacity' 直接修改！
}
.popup {
  opacity: 0.8;
}

    似乎也就是這些了，但是我現在用起來這個東西還是心驚膽戰的，真害怕它突然崩掉了……

    我之前説過我的博客不會設置社交功能，但是我現在其實是想去設置一個評論區的，或者設置一個瀏覽量統計的功能。説實話，我覺得前者可能性稍小，後者可能性較大，但都不是今天。
