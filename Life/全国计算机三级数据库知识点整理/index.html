    DBAS 开发方法

    数据库基本概念

    • 数据：临时性数据、持久性数据，型、值
    • 数据库 DB
    • 数据库系统 DBS：数据库、数据库管理系统、数据库管理员、硬件软件平台
    • 数据库应用系统 DBAS：数据库系统、应用软件、应用界面

    软件工程与数据库技术

    • 软件工程三要素：方法、工具、过程
    • 软件生命周期：定义、开发、使用与维护、退役
    • 软件开发模型：瀑布模型、快速原型模型、螺旋模型

    DBAS 生命周期

    • 五个基本活动：项目规划、需求分析、系统设计、实现与部属、运行管理与维护
    • 三条设计主线：数据组织与存储设计（设计数据库）、数据访问与处理设计（设计数据库事务）、应用设计（设计应用程序）
    • 三个步骤：概念设计、逻辑设计、物理设计
    规划与分析
    • 系统规划与定义：……确定范围和边界、确定任务视图
    • 可行性分析：技术、经济、操作可行性，开发方案选择
    • 项目规划
    需求分析（干什么）
    • 最困难、最耗费时间
    • 数据、功能（做什么）、性能（做到什么程度）、其他（存储、安全性、备份与恢复）
    系统设计（怎么干）
    • 概念设计：数据库概念模型设计、系统总体设计
    • 逻辑设计：数据库逻辑结构设计、数据库事务概要设计、应用程序概要设计
    • 物理设计：数据库物理结构设计，数据库事务详细设计、应用程序详细设计

    需求分析

    • 深入描述软件的功能与性能
    • 在系统分析人员与用户不断交互过程中完成的
    • 需求获取的方法：面谈、实地观察、问卷调查、查阅资料
    • 需求分析（获取→分析）的过程：表示问题、建立需求模型、描述需求、确认需求

    需求分析方法

    • 结构化分析与建模方法：DFD（数据流图）建模、IDEF 建模

      • 对数据的加工（处理）
      • 数据存储
    • 过程：顶层 DFD 图、第一层 DFD 分解图、DFD 层次结构图

    数据库结构设计

    数据库概念设计（核心环节）

    • 依据：需求分析的结果（包括 DFD 图）
    • 过程
      • 明确建模目标
      • 定义实体集、联系
      • 建立信息模型
      • 确定实体集属性
      • 对信息模型进行集成与优化
    • 结果：概念模型（ER）
    ER 建模方法
    • E-R 模型（实体联系模型）、E-R 图
    • 概念：实体/实例（实体集）、属性（域）、码/键（主码）、联系（联系的属性）
    • 三类联系：一对一联系（1:1）、一对多联系（1:n）、多对多联系（m:n）

    数据库逻辑设计

    • 任务：将概念模型（如 ER 图）转化为 DBMS 支持的数据模型（如关系模型）
    • 三种数据模型：层次模型、网状模型、关系模型√
    数据依赖
    • 函数依赖
      • 平凡函数依赖、非平凡函数依赖（因变量是否是自变量的子集）
      • 完全函数依赖、部分函数依赖（因变量否是只是自变量子集的函数）
      • 传递函数依赖
    • 多值依赖

    码：候选码、主码、外码

    数据规范化
    • 1NF：所有属性都不可再分
    • 2NF：所有非主属性完全函数依赖与主码
    • 3NF：所有非主属性都不传递依赖与主码
    • BNF：消除主属性对码的部分和传递函数依赖
    • 4NF：消除非平凡且非函数依赖的多值依赖
    数据库逻辑模型的产生（数一数至少有几个关系模式）
    • 一个实体转化成一个关系模式
    • 一个1:1联系可以独立，也可以和任意一端合并
    • 一个1:n联系可以独立，也可以与n端合并
    • 一个m:n联系转换为一个关系模式
    • 同一实体集的实体间的联系同上
    • 多元联系转换为一个关系模式

    数据库物理设计

    • 设计数据库的存储结构和物理实现
    • 物理结构
      • 物理设备上的存储结构与存取方法
      • 以文件形式存储在外设存储介质上
      • 文件组织、文件结构、文件存取、索引技术
    索引
    • 提高性能，付出存储空间
    • 分类
      • 有序索引（数据文件采用顺序文件结构）
        • 聚集索引（索引项与数据记录排列顺序一致、索引顺序文件）、非聚集索引
        • 稠密索引、稀疏索引（存在查找码没有索引）
        • 主索引（主码属性集上的索引）、辅索引（非主属性上的索引）
        • 唯一索引（索引不包含重复值）
        • 单层索引、多层索引（大数据量文件）
      • 散列索引
    五个环节
    1. 描述逻辑模式：T-SQL、文件结构（堆文件、顺序文件、聚集文件、索引文件、散列文件）
    2. 文件组织与存取设计
      • 易变部分与稳定部分、高频部分与低频部分
      • 表和索引放在不同磁盘上
      • 影响数据文件存储结构的因素：存取时间、存储空间利用率、维护代价
        • 解决办法：适当冗余、增加聚簇功能
      • 常用存取方法：索引方法、聚簇方法、HASH 方法
      • 建立索引的原则
        • 经常在操作条件、连接条件、聚集函数参数中出现
        • 建立聚集索引的原则：所谓排序、分组、限制条件，重复性较大
    3. 数据分布设计：水平划分、垂直划分
    4. 确定系统配置
    5. 物理模式评估：时间和空间效率

    应用系统安全架构设计

    数据安全设计
    1. 安全性保护
      • Windows 身份验证、SQL 身份验证
      • 视图机制
    2. 完整性保护
      • 正确性、一致性、相容性
      • 约束条件：实体完整性、参考完整性、用户自定义完整性
      • 三种级别：列（类型、范围、精度、排序）、元组（一条记录中各属性之间的联系约束）、关系（若干记录间、关系集合与集合之间的联系）
      • SQL Server 五种约束类型：主键约束、外键约束、唯一性约束、缺省约束、检查约束
    3. 并发性控制
      • 写锁（X 锁、排它锁）：别的事务不能再加锁、读取、修改
      • 读锁（S 锁、共享锁）：别的事物只能加 S 锁、读取，不能加 X 锁、修改
      • 避免活锁：先来先服务
      • 避免死锁
        • 按照同一顺序访问资源
        • 避免事务交互性
        • 小事务模式，缩短长度和占用时间
        • 使用行锁（记录级别的锁），少用表级别的锁
        • ？绑定链接
    4. 数据备份与恢复
      • 原理：其他存储器上的冗余数据
      • 策略：双机热备、数据转储、数据加密存储（高敏感数据）
    5. 数据加密传输
    环境安全设计
    • 漏洞与补丁、计算机病毒防护、网络环境安全、物理环境安全
    制度安全设计

    数据库及数据库对象

    创建及维护数据库

    1. SQL Server 数据库分类
      • 系统数据库：master、msdb、tempdb（临时）、model（模板）、resource
      • 用户数据库
    2. SQL Server 数据库 组成
      • 数据文件
        • .mdf 主要数据文件：一个，不小于 3MB
        • .ndf 次要数据文件：几个都行，可在多个磁盘存放
      • 日志文件
        • .ldf 事务日志文件：至少一个
        • 两种：以记录为单位、以数据块为单位
      • 存储空间分配
        • 最小单位：数据页（1页8KB）
        • 行不能跨页存储
    3. 文件组：主文件组、用户定义文件组
      • 日志文件不在文件组中，日志与数据分开管理
      • 一个文件只能在一个文件组中，且不能移动到其他文件组
      • 只能指定一个默认文件组
    4. 文件属性：文件名（逻辑名）及其位置（物理名）、初始大小（不小于 model 数据库主要数据文件大小）、增长方式、最大大小

    安全管理

    安全控制概述

    • 安全性：防止非法用户故意破坏
    • 完整性：防止合法用户无意破坏
    • 四个阶段：身份验证、操作权限控制、文件操作控制、加密存储与冗余

    存取控制

    自主存取控制
    • 自主安全模式，GRANT REVOKE DENY
    • 用户分类：系统管理员、数据库对象拥有者、普通用户
    强制存取控制
    • 敏感度标记
      • 主体：许可证级别
      • 客体：绝密、秘密、可信、公开
      • 主体大于等于客体时可读，主体等于客体时可写

    存储空间管理

    性能优化

    1. 运行环境与参数调整：CPU、内存、磁盘、网络
    2. 模式调整与优化：反规范化
      • 增加派生性冗余列：由计算生成
      • 分割表：水平分割（Union）、垂直分割（Join）
      • 增加冗余列、重新组表、新增汇总表
    3. 存储优化
      • 物化视图（索引视图）
      • ？聚集
    4. 查询优化
      • 合理使用索引
        • 建立索引：经常在查询中被作为条件使用、频繁排序或分组、值域很大
        • 更新大量数据后，删除并重建索引
      • 避免或简化排序
      • 消除对大型表数据的顺序存取
      • 避免复杂正则表达式
      • 使用临时表加速查询
      • 少使用左/右/外连接，多使用内连接
      • 存储过程尽量使用自带返回参数而非自定义返回参数
      • 不要随意使用游标

    故障管理

    概述

    • 故障类型和解决方案
      1. 事务内部故障
        • 预期的（大部分）：事务回滚
        • 非预期的：系统自动恢复
      2. 系统故障（软故障）
        • 软件漏洞、突然断电等导致数据不一致
        • UNDO、REDO
      3. 介质故障（硬故障）
        • 物理存储设备损坏，数据文件和数据全部丢失
        • 容错策略：软件容错、硬件容错
          • 软件容错：数据库备份、事务日志文件
          • 硬件容错：双物理存储设备、两套相同的数据库系统同步
      4. 计算机病毒故障
        • 防火墙、杀毒软件、数据库备份文件
    • 恢复技术
      • 基本原理是冗余
      • 数据备份、登记日志文件

    数据转储

    • 静态转储和动态转储、日志文件
    • 转储机制
      • 完全转储
      • 增量转储：上次转储之后
      • 差量转储：上次完全转储之后

    日志文件

    • 动态转储必须建立日志文件，静态转储可选
    • 以记录为单位、以数据块为单位
    • 先写日志文件后写数据库

    硬件容错方案

    • 磁盘保护技术 RAID
      • RAID-0：多个磁盘合并，无冗余，并行输入输出，最快
      • RAID-1：磁盘相互作镜像，可靠性最高
      • RAID-10：前两者合体，冗余度为 50%
      • RAID-5：至少三个硬盘
    • 服务器容错技术
      • 双机热备、心跳检测
    • 数据库镜像与容灾
      • 双机互备援模式、双机热备份模式
    • SQL Server 数据库镜像
      • 镜像的复制不能直接访问
      • 三种实现方式
        • 高可用性：两台服务器同步事务写入，自动错误恢复
        • 高保护性：两台服务器同步事务写入，手动错误恢复
        • 高性能：两台服务器写入不同步，手动错误恢复

    备份与恢复数据库

    恢复模式

    1. 简单恢复模式：简略地记录大多数事务，不备份事务日志
    2. 完整回复模式：完整地记录了所有的事务，并保留所有的事物日志记录
    3. 大容量日志恢复模式：简略地1记录大多数大容量操作，完整地记录其他事务
    4. 系统数据库：修改之后进行备份；用户数据库：周期性备份
    5. 立即备份的情况：建立数据库或批量加载数据、创建索引、清理事务日志、执行大容量数据操作

    备份机制

    • 备份类型：完整数据库备份、差异数据库备份、文件备份、差异文件备份、事务日志备份
      • 事务日志备份仅用于完整恢复模式或大容量日志恢复模式
      • 三种事务日志备份：纯日志备份、大容量操作日志备份、结尾日志备份
    • 常用备份策略
      • 完整备份
      • 完整备份 + 日志备份
      • 完整备份 + 差异备份 + 日志备份
    • 还原的顺序：完全备份→差异备份→日志备份

    大规模数据库架构

    分布式数据库

    • 最基本特征：本地自治、非集中式管理、高可用性
    • 数据分布策略
      • 数据分片：水平、垂直、导出、混合
      • 数据分配：集中式、分割式、全复制式、混合式
    • 分布透明性：分片透明性、位置透明性、局部数据模型透明性

    并行数据库

    • 三种硬件结构
      • 共享内存系统（可能内存冲突）
      • 共享磁盘系统（增加了通信代价）
      • 无共享资源系统（大型并行数据库系统最优结构）
      • 层次结构（前三者结合）
    • 一维数据划分
      • 轮转法：有效应用于需要访问整个关系的查询处理
      • 范围（划分）法：可能会导致数据偏斜
      • 散列法：能保持均匀分布

    云计算数据库

    • 云计算服务类型：基础设施作为服务（IaaS）、平台作为服务（PaaS）、软件作为服务（SaaS）
    • 云：公共云、私有云、混合云
    • 谷歌云：BigTable

    XML 数据库

    • XML：可扩展标识语言
    • SQL Server 语句 FOR XML RAW

    数据仓库与数据挖掘

    • 分析型数据、操作型数据（OLTP）

    数据仓库技术

    • 数据仓库的特性：面向主题、集成、不可更新（以批量方式处理）、时间特性/随时间变化
    • 细化程度与粒度
    • 元数据：技术型元数据和业务型元数据
    • 操作性数据存储 ODS
      • 四个基本特点：面向主题的、集成的、可变的、当前或接近当前的
      • 四类：更新频率秒级、小时级、天级，根据数据来源方向和类型区分

    联机分析处理与多维数据模型

    • 多维分析：钻取与卷起、切片和切块、旋转

    数据挖掘技术

    • 三个阶段：数据准备（数据选取、数据预处理、数据变化），数据挖掘，结果解释评估
    • 分类、聚类、时间序列
